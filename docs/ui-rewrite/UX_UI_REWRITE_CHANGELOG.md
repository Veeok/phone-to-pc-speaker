# UX/UI Rewrite Changelog

| File | What Changed | Why It Changed | Audit Pattern Mapping |
| --- | --- | --- | --- |
| `apps/desktop/src/renderer/src/App.tsx` | Rebuilt the renderer into a centered operator shell with hero controls, supporting side panels, floating launcher, command palette, settings modal, readiness guide, toast/banner feedback, and staged confirmations. | The old single-screen dashboard preserved logic but not the target shell architecture or interaction model. | Single primary surface, floating action launcher, command palette, settings shell, guided setup, global feedback, progressive disclosure, confirmation flow. |
| `apps/desktop/src/renderer/src/styles.css` | Replaced the previous dashboard styling with a token-based dark control-room system, responsive shell layout, luminous borders, motion rules, component variants, and overlay styling. | The rewrite needed a unified visual system instead of section-specific styling. | Dark-first premium aesthetic, tokenized design system, rounded cards, border glow, strong spacing rhythm, responsive stacked panels. |
| `apps/desktop/src/renderer/src/view-models.ts` | Added richer UI view-model helpers for setup progress, banner state, device search/sort, connection progress, and relative timestamps while preserving domain truth. | The new shell requires derived presentation models without changing backend behavior. | Strong information hierarchy, hidden logic preservation, readiness/setup adaptation, richer row/card item behavior. |
| `apps/desktop/src/renderer/src/ui.tsx` | Added reusable primitives for panels, badges, buttons, cards, modals, settings rail, inputs, toggles, toasts, banners, skeletons, detail blocks, progress bars, and disclosures. | The rewrite needed reusable experience patterns instead of one-off markup. | Component standardization, reusable design language, consistent interaction states. |
| `apps/desktop/src/renderer/src/CommandPalette.tsx` | Added searchable command/action overlay with grouped results, keyboard navigation, empty state, and device shortcuts. | Fast access was part of the target interaction architecture and did not exist before. | Command/search palette, low cognitive load workflow acceleration, secondary tasks in overlay. |
| `apps/desktop/src/renderer/src/preferences.ts` | Added renderer-local UI settings persistence for appearance, workflow safety, diagnostics defaults, and accessibility behavior. | The new settings shell needed real state backing without changing backend APIs. | Settings shell with category rail, hidden UX rules, motion preferences, progressive disclosure. |
| `apps/desktop/src/renderer/src/icons.tsx` | Added a consistent icon set for shell navigation, workflow actions, readiness, and diagnostics. | The rewrite needed reusable iconography to reinforce status and control grouping. | Rich control grouping, readable operator-console feel, state-forward affordances. |
| `apps/desktop/src/renderer/src/utils.ts` | Added shared class-name composition helper. | Shared primitives and palette rows needed consistent utility composition. | Reusable component foundation. |
| `docs/ui-rewrite/UX_UI_REWRITE_PLAN.md` | Documented current architecture, target architecture, mapping, implementation order, and risks. | The rewrite required an explicit plan artifact in the requested report directory. | Analyze -> plan phase deliverable. |
| `docs/ui-rewrite/DESIGN_SYSTEM_SPEC.md` | Documented tokens, typography, spacing, component variants, motion, responsiveness, and accessibility rules. | The rewrite required a design-system spec alongside the implementation. | Design token plan, reusable visual language. |
| `docs/ui-rewrite/UI_COMPONENT_INVENTORY.md` | Added component-by-component inventory with purpose, inputs, states, and usage. | The rewrite required a reusable primitive inventory. | Component standardization deliverable. |
